# Contributing to Requiems API

Thanks for your interest in contributing! We welcome improvements from the
community and appreciate your help making this project better.

## What We Accept

We're happy to review contributions including:

- **Bug fixes** – Help us squash issues
- **Performance improvements** – Make it faster and more efficient
- **Documentation** – Clarify usage, add examples, fix typos
- **Features** – New functionality that aligns with project goals
- **Architecture discussions** – Ideas for improving the codebase structure

## How to Contribute

1. **Fork** the repository
2. **Create a branch** for your changes (`git checkout -b feature/your-feature`)
3. **Make your changes** and commit with clear messages
4. **Submit a pull request** with a description of what you've changed and why

We'll review PRs as quickly as possible and provide constructive feedback.

## License Note

## Questions?

Open an issue or reach out – we're here to help!
